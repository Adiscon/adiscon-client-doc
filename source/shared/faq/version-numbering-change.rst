Why did product versions jump to 26.07?
=======================================

Question
--------

Why is the current version ``26.07`` when older product versions used numbers
such as 18, 19, 15, or 8?

Answer
------

Adiscon changed to calendar-based version numbers. In ``26.07``, ``26`` means
2026 and ``07`` means July. There are no missing intermediate product releases.

Details
-------

The latest old-style version families were:

- **WinSyslog**: ``18.x``
- **EventReporter**: ``19.x``
- **MonitorWare Agent**: ``15.x``
- **rsyslog Windows Agent**: ``8.x``

The new ``YY.MM`` format starts with ``26.07``.
